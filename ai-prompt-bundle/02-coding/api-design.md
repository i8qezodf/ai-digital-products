# API设计助手提示词

## 模板

```
请为以下场景设计RESTful API：

## 业务场景
[描述业务需求，如：用户管理系统]

## 功能需求
1. [功能1]
2. [功能2]
3. [功能3]

## 技术栈
- 语言：[如 Java/Node.js]
- 框架：[如 Spring Boot/Express]
- 数据库：[如 PostgreSQL/MongoDB]

## 请生成

### 1. 资源建模
- 主要资源列表
- 资源之间的关系
- 资源字段定义

### 2. API端点设计

请用以下格式：

| 方法 | 端点 | 描述 | 请求体 | 响应 |
|-----|------|-----|-------|------|
| GET | /users | 获取用户列表 | - | User[] |
| POST | /users | 创建用户 | User | User |
| GET | /users/{id} | 获取单个用户 | - | User |
| PUT | /users/{id} | 更新用户 | User | User |
| DELETE | /users/{id} | 删除用户 | - | 204 |

### 3. 数据模型

```[数据库类型]
// 如 MongoDB schema 或 SQL schema
```

### 4. 错误处理

| 错误码 | 描述 | 处理方式 |
|-------|------|---------|
| 400 | Bad Request | 参数验证失败 |
| 401 | Unauthorized | 未认证 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 500 | Server Error | 服务器错误 |

### 5. 认证方案
- 建议的认证方式
- Token格式
- 刷新机制

### 6. 分页/过滤/排序
- 分页参数设计
- 过滤字段
- 排序选项

### 7. 示例代码

[请求/响应示例]
```

## 使用说明

1. 详细描述业务场景
2. 列出所有功能需求
3. 指定技术栈便于给出最佳实践
4. 可要求生成特定语言的代码

## 适用场景

- 新项目API设计
- 现有API重构
- 微服务拆分
- 移动端API设计
