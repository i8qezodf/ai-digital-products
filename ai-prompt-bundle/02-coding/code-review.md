# 代码审查提示词

## 模板

```
请审查以下代码，重点关注：

语言/框架：[如 Python/Django]
代码功能：[代码目的]

## 代码：
```
[粘贴代码]
```

## 审查维度

请从以下角度审查：

### 1. 正确性
- 是否有逻辑错误？
- 边界条件是否处理？
- 是否有潜在的bug？

### 2. 安全性
- 是否有SQL注入风险？
- 是否有XSS漏洞？
- 敏感数据是否泄露？
- 权限控制是否正确？

### 3. 性能
- 是否有不必要的循环？
- 数据库查询是否优化？
- 是否有内存泄漏风险？

### 4. 代码风格
- 命名是否清晰？
- 注释是否充分？
- 是否符合该语言的最佳实践？

### 5. 可维护性
- 模块化是否合理？
- 依赖是否清晰？
- 测试覆盖是否足够？
