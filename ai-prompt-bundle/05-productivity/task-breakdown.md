# 任务分解器（WBS）

## 模板

```
请将以下任务分解为详细的工作分解结构（WBS）：

## 任务信息

**项目名称：**
[项目名]

**最终目标：**
[要达成的目标]

**截止日期：**
[日期]

**可用资源：**
- 人力：[人数/角色]
- 预算：[金额]
- 工具：[可用工具]

**约束条件：**
[任何限制或特殊要求]

## 请生成

### 1. 顶层分解（3-5个主要阶段）

```
1. [阶段名]
   1.1 [子阶段]
       1.1.1 [具体任务]
       1.1.2 [具体任务]
   1.2 [子阶段]
2. [阶段名]
   ...
```

### 2. 任务详情表

| 序号 | 任务名称 | 所属阶段 | 负责人 | 预计工时 | 依赖任务 | 优先级 |
|-----|---------|---------|-------|---------|---------|-------|
| 1 | | | | Xh | - | 🔴 |
| 2 | | | | Xh | 1 | 🟡 |

### 3. 里程碑清单

| 里程碑 | 定义标准 | 目标日期 | 状态 |
|-------|---------|---------|------|
| M1 | | | ⬜ |
| M2 | | | ⬜ |

### 4. 资源分配表

| 资源/角色 | 分配比例 | 可用时间 | 备注 |
|----------|---------|---------|------|
| | | | |

### 5. 风险识别

| 风险 | 影响 | 概率 | 应对策略 |
|-----|------|------|---------|
| | | | |

### 6. 时间线估算

- 总工期：X天/X周
- 关键路径：
- 浮时：

### 7. 执行建议

- 执行顺序建议
- 需要重点关注的任务
- 可能的瓶颈
```

## 使用说明

1. 目标描述要具体可衡量
2. 分解到最小可执行任务
3. 明确任务间的依赖关系
4. 预留应急时间（建议10-20%）

## WBS分解原则

- 每层分解出3-7个子项
- 每个叶子任务2-40小时完成
- 任务描述使用动词开头
- 避免模糊任务（如"相关工作"）
