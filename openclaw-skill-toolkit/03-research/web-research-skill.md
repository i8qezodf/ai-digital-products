# Web Research Skill

Gather and summarize information from the web efficiently.

## When to Use

You need to learn about a topic, compare options, or gather background information quickly. Use when you have a specific research question.

## The Skill

```
Research this topic:

RESEARCH QUESTION: [what you need to know]
DEPTH: [quick overview / thorough analysis]
SOURCES NEEDED: [minimum number or specific types]

Process:

1. SEARCH STRATEGY
   - What to search for
   - Alternative search terms
   - Sources to prioritize (docs, forums, reviews, etc.)

2. INFORMATION GATHERED
   For each source:
   - Source: [URL or type]
   - Key information: [bullet points]
   - Credibility: [high/medium/low]
   - Relevance: [directly answers / related / tangential]

3. SYNTHESIS
   - What the consensus is (or where opinions differ)
   - Key takeaways
   - What the sources don't tell you
   - Contradictions or outdated info

4. ANSWER
   Direct answer to your research question based on findings

5. FURTHER QUESTIONS
   - Follow-up research needed
   - Things that are unclear
   - Topics that got cut off
```

## Example

```
RESEARCH QUESTION: What's the best way to send emails from a web app in 2024?
DEPTH: Thorough analysis
SOURCES NEEDED: At least 5

SEARCH STRATEGY:
- Primary: "transactional email service comparison 2024"
- Alternatives: "best email API for developers", "SendGrid vs Mailgun vs AWS SES"
- Prioritize: Developer documentation, comparison blog posts

SYNTHESIS:
- Top services: SendGrid, Mailgun, AWS SES, Resend
- SES is cheapest at scale but most complex
- SendGrid has best developer experience
- Resend is newer, modern API, gaining traction

ANSWER:
For most apps: Start with SendGrid or Resend. SES if cost is critical and you have DevOps capacity.

FURTHER:
- If choosing Resend: check deliverability rates
- Email warmup process needed for new domains
```

## Tips

- Be specific in your research question — broad questions get shallow answers
- Cross-reference claims across multiple sources
- Note the date — information can be outdated
- Look for the "why" not just the "what"

## Variations

**Competitive research:** Add comparison matrix, pricing analysis
**Product review:** Add pros/cons table, user sentiment
**Technical research:** Include code examples, benchmarks
**Market research:** Add trend data, audience size, growth
