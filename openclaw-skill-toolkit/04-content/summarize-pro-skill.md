# Summarize Pro Skill

Summarize anything — articles, meetings, documents, videos — in the format you need.

## When to Use

You have long content and need to extract what matters. Articles, meeting transcripts, reports, emails, YouTube videos, PDFs.

## The Skill

```
Summarize this content:

TYPE: [article / meeting / email / report / video / document / other]
LENGTH: [short (3-5 bullets) / medium (1 paragraph) / long (detailed)]
FOCUS: [everything / key points / action items / specific topic]

SOURCE:
[paste content or describe source]

Generate summary using the most appropriate format:

FOR QUICK OVERVIEW:
## TL;DR
[One sentence summary]

## Key Points
- [Point 1]
- [Point 2]
- [Point 3]

## What Stands Out
[One interesting observation]

FOR DETAILED SUMMARY:
## Summary
[2-3 paragraph overview]

## Key Points
- [Main point 1] + [why it matters]
- [Main point 2] + [why it matters]
- [Main point 3] + [why it matters]

## Notable Details
[Interesting specifics worth remembering]

## Questions / Uncertainties
[Things that weren't clear or need follow-up]

FOR ACTION ITEMS:
## Action Items
- [ ] [Specific action 1] — who/when if mentioned
- [ ] [Specific action 2]
- [ ] [Specific action 3]

## Decisions Made
- [Decision 1]
- [Decision 2]

## Open Questions
- [Question 1]
- [Question 2]

FOR SPECIFIC TOPIC:
[Re-summarize focusing only on [topic], filtering out everything else]
```

## Example

**Input:**
```
TYPE: Meeting transcript
LENGTH: Medium
FOCUS: Action items and decisions
SOURCE: [transcript pasted]
```

**Output:**
```
## Decisions Made
- Launch date set for June 15
- Budget approved at $8k (reduced from $12k request)
- Using in-house design team, not agency

## Action Items
- [ ] Sarah: Send updated timeline by Friday
- [ ] Dev team: Confirm technical feasibility by next Monday
- [ ] Mike: Get vendor quotes by Wednesday
- [ ] Everyone: Review proposal doc before Thursday sync

## Open Questions
- What's the fallback if design team is unavailable?
- Do we need board approval for the $8k?
```

## Tips

- Different content needs different summary types — match to use case
- For meetings: prioritize decisions and actions over discussion
- For articles: consider what you'd want to remember in 6 months
- If summary doesn't feel right, try a different focus angle

## Variations

**Chapter summary:** For books, courses, video series
**Comparison summary:** For alternatives being evaluated
**Translation summary:** For non-native content
**Executive brief:** For stakeholders who need high-level view
