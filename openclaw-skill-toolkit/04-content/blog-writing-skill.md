# Blog Writing Skill

Generate blog posts that people actually read.

## When to Use

Creating blog content — technical posts, tutorials, opinion pieces, or thought leadership.

## The Skill

```
Write a blog post:

TOPIC: [what it's about]
ANGLE: [specific hook or unique perspective]
TARGET: [who you're writing for]
LENGTH: [short ~500 words / medium ~1000 / long ~2000]
STYLE: [tutorial / opinion / case study / listicle / personal story]
TONE: [conversational / professional / humorous / authoritative]

KEY POINTS TO COVER:
[paste or list main points]
SOURCE MATERIAL: [if you have source docs, paste them]

FORMAT:
## Title
[SEO-friendly, compelling title]

## Opening Hook
[First paragraph that makes someone keep reading]

## Main Content
[Organized with headers, flows naturally]

## Call to Action
[What you want readers to do after]

FOR TECHNICAL POSTS, ADD:
- Code examples (working, not pseudocode)
- Screenshots/visuals described
- Prerequisites mentioned
- Common mistakes to avoid

FOR OPINIONS, ADD:
- Clear thesis in first two paragraphs
- Address counterarguments
- Personal experience/examples
- Why this matters now

FOR TUTORIALS, ADD:
- Step-by-step numbered approach
- Time estimates
- Prerequisites
- Troubleshooting section
```

## Example

```
TOPIC: Using AI to review your code
ANGLE: Practical guide, not hype
TARGET: Developers curious about AI tools
LENGTH: Medium (~1000 words)
STYLE: Tutorial
TONE: Conversational, peer-to-peer

OUTPUT:

## Stop Writing Code Reviews Manually — AI Does Them Faster

I used to spend 2 hours on code reviews. Now I spend 20 minutes — and catch more bugs.

Here's what changed.

[Full tutorial follows with working code examples]
```

## Tips

- Hook in the first paragraph — nobody reads past a boring opener
- Break up long posts with headers, bullets, images
- Code examples should actually run
- End with something to do — "try this" beats "that's it"
- Read it out loud before publishing — you'll catch awkwardness

## Variations

**Thread format:** For Twitter/X threading
**Video script:** Add timing, visual cues, pacing notes
**Newsletter issue:** Match the format of your email series
**LinkedIn post:** Shorter, more professional, call to action in comments
