# Decision Framework Skill

Make structured decisions when you're stuck or the stakes are high.

## When to Use

You have a decision to make and want to think it through systematically. Use when the choice is important, reversible decisions don't need this.

## The Skill

```
Help me make this decision:

THE DECISION: [what you need to decide]
DECIDED BY: [deadline or when you need this]
STAKES: [what's the worst that happens if you choose wrong]

OPTIONS:
1. [Option A]
   - What it is: [brief description]
   - Best case: [ideal outcome]
   - Worst case: [how bad could it get]
   - Effort: [low/medium/high]
   - Confidence: [how sure you are about this choice]

2. [Option B]
   ...

3. [Option C or "do nothing"]
   ...

COMPARISON MATRIX:
                  | Option A | Option B | Option C
------------------+----------+----------+----------
Outcome quality   |          |          |
Reversibility     |          |          |
Effort required   |          |          |
Time to result    |          |          |
Confidence        |          |          |

RECOMMENDATION:
[Based on the above, what makes sense and why]

REVERSIBILITY TEST:
If I'm wrong, how hard is it to change course?
- Easy: [option can be undone]
- Hard: [significant cost to reverse]
- Permanent: [no going back]

What would have to be true for each option to be the right call?
```

## Example

```
THE DECISION: Should I publish my newsletter publicly or keep it invite-only to start?
DECIDED BY: Before next issue (3 days)
STAKES: Public means more growth potential but also more pressure; invite-only means slower growth but more control

OPTIONS:
1. Public immediately
   - What: Open Substack, share link freely
   - Best case: Rapid subscriber growth,意外 virality
   - Worst case: Low initial numbers, discouraging
   - Effort: Low
   - Confidence: 60%

2. Invite-only for first month
   - What: Only share with personal network
   - Best case: 50 warm subscribers, refine content
   - Worst case: Stalls at network size
   - Effort: Medium
   - Confidence: 70%

3. Hybrid: Semi-public
   - What: Public archive, subscribe gate
   - Best case: Best of both worlds
   - Worst case: Confuses positioning
   - Effort: Medium
   - Confidence: 50%

RECOMMENDATION: Option 2 (invite-only) because first issue quality matters more than quantity, and you'll learn what resonates with a smaller group before going wide.

REVERSIBILITY TEST: Easy — you can always open up later.
```

## Tips

- Don't overthink reversible decisions — pick and adjust
- The "do nothing" option is often underrated
- If you're stuck between similar options, flip a coin — then notice which outcome you hoped for
- Reversibility matters more than people think

## Variations

**Team decisions:** Add stakeholder buy-in row to matrix
**Technical decisions:** Add maintainability, scalability columns
**Hire/not hire:** Add "try for N months" option where possible
