# Task Breakdown Skill

Turn vague goals into concrete, actionable steps.

## When to Use

You have something you want to accomplish but aren't sure what to do first. The task feels too big or unclear. You need a structured plan.

## The Skill

```
Analyze this task and break it down into actionable steps:

TASK: [what you're trying to do]
DEADLINE: [if any]
CONSTRAINTS: [budget, tools, skills you have]
UNKNOWN AREAS: [anything you're unsure about]

Break it down following this structure:

1. CLARIFY — What does success look like? What's the minimum viable version?
2. IDENTIFY — What are the dependencies? What must be done before other things?
3. SEQUENCE — Put everything in order. What's blocking what?
4. ESTIMATE — How long should each step take?
5. PRIORITIZE — Which 20% of tasks deliver 80% of results?

For each step provide:
- What to do (specific action)
- How to know it's done (clear criteria)
- Potential blocker and how to handle it

Output as a numbered list with estimated time per item.
```

## Example

**Input:**
```
TASK: Launch a newsletter
DEADLINE: 30 days
CONSTRAINTS: $0 budget, no email platform chosen
UNKNOWN AREAS: Content strategy, audience building
```

**Output would include:**
1. Define niche and target reader (2-3 hours)
2. Choose platform (Substack, Ghost, etc.) (1 hour)
3. Write 5 sample issues (1-2 days)
4. Set up email capture and landing page (3-4 hours)
5. Create social promotion plan (2 hours)
6. Launch day checklist

## Tips

- If a step takes more than 4 hours, break it further
- Group similar tasks together
- Build in review points between major phases
- Start with the "ugly first" — do the hard parts early

## Variations

**For creative projects:** Add a mood board / inspiration phase before breaking down
**For technical projects:** Add an architecture spike / research phase first
**For team projects:** Add stakeholder alignment step at the start
